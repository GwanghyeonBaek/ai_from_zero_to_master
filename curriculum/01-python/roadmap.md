## 2) 학습 목표 (Learning Outcomes)

완료 시 학습자는 다음을 할 수 있어야 한다.

1. 기초 문법으로 문제를 작은 함수 단위로 분해해 해결한다.
2. 리스트/딕셔너리/셋/튜플을 문제 성격에 맞게 선택한다.
3. `for`/`while`/컴프리헨션을 상황에 맞게 사용한다.
4. 예외처리와 입력검증으로 안정적인 코드를 작성한다.
5. 파일(CSV/TXT) 읽기/쓰기 자동화 스크립트를 만든다.
6. 모듈 구조를 나눠 재사용 가능한 코드로 정리한다.

## 6) 자주 나는 실수 & 교정법

1. **문법만 보고 손코딩을 안 함**
   - 교정: 하루 최소 30줄 직접 입력
2. **한 파일에 모든 코드 몰아넣기**
   - 교정: 함수/모듈 분리 규칙 강제
3. **예외처리 생략**
   - 교정: 입력 받는 코드에는 기본 `try/except` 추가
4. **에러 메시지 읽지 않고 감으로 수정**
   - 교정: 에러 원인 1문장 기록 후 수정

## 9) 외부 공신력 자료 액기스 반영 (Python)

### 참고자료
- Python 공식 튜토리얼: https://docs.python.org/3/tutorial/
- Automate the Boring Stuff: https://automatetheboringstuff.com/
- Real Python: https://realpython.com/

### 핵심 추출 내용(액기스)
1. **작게 쪼개기(함수 단위)**: 긴 스크립트보다 작은 함수 조합이 디버깅/재사용에 유리.
2. **입력 검증 우선**: 타입/범위 검증을 먼저 넣으면 예외가 빨리 드러남.
3. **파일 처리 자동화**: 반복 수작업은 스크립트로 바꿔야 실무 생산성이 올라감.
4. **표준 라이브러리 우선**: `pathlib`, `csv`, `json`만으로도 다수 실무 문제 해결 가능.
