# 04-data-viz — 완주형 로드맵 (실무형 데이터 시각화)

> 목표: "예쁜 차트"가 아니라 **의사결정을 바꾸는 차트**를 만든다.

---

## 1) 챕터 완료 기준 (Definition of Done)

아래를 모두 충족하면 04 완료:

- [ ] 차트 선택 근거를 데이터 질문과 함께 설명 가능
- [ ] Matplotlib/Seaborn 기반 시각화 12개 이상 직접 구현
- [ ] 왜곡된 시각화 5개 이상 교정
- [ ] 시각화 리포트 2건(각 1~2페이지) 제출
- [ ] 미니 프로젝트 2개(대시보드형 + 스토리형) 완료
- [ ] 루브릭 75점 이상 + 회고 제출

---

## 2) 학습목표 (Learning Outcomes)

1. 분석 질문을 시각화 질문으로 변환한다.
2. 비교/추세/분포/관계에 맞는 차트를 선택한다.
3. 축 범위·색상·주석으로 해석 오류를 줄인다.
4. 시각화 결과를 비즈니스 문장으로 보고한다.
5. 재현 가능한 시각화 코드(입력→처리→출력)를 작성한다.

---

## 3) 주차별 커리큘럼 (3주)

### Week 1 — 차트 문법과 읽기 쉬운 기본기
- 비교: 막대/점그래프
- 추세: 선그래프, 이동평균
- 분포: 히스토그램/박스플롯
- 실습: 같은 데이터를 3방식으로 시각화하고 장단점 비교

### Week 2 — 고급 표현과 시각적 설계
- 관계: 산점도/회귀선
- 패턴: 히트맵/페싯(facet)
- 시각적 인코딩(색/크기/형태), 접근성(색약 친화)
- 실습: 잘못된 차트 교정, 대시보드 초안 제작

### Week 3 — 스토리텔링과 리포팅
- 한 장표 스토리 구조(질문→증거→결론→액션)
- 주석/하이라이트/레이아웃
- 미니 프로젝트 A/B 완성 + 발표

## 4) 필수 과제 (12개)

- [ ] 카테고리 매출 비교(막대)
- [ ] 월별 추세(선 + 이동평균)
- [ ] 분포 비교(히스토그램 + 박스플롯)
- [ ] 산점도 + 회귀선
- [ ] 상관 히트맵
- [ ] 페싯으로 세그먼트 비교
- [ ] 축 왜곡 차트 교정
- [ ] 색상 접근성 개선 버전 제작
- [ ] 주석 포함 핵심 인사이트 차트
- [ ] 1페이지 시각화 리포트
- [ ] 미니 프로젝트 A (운영 대시보드)
- [ ] 미니 프로젝트 B (의사결정 스토리 리포트)

## 5) 평가 기준 (100점)

- 질문-차트 적합성: 25

- 시각적 품질(가독성/레이아웃): 20
- 스토리텔링(결론/액션 제안): 20
- 재현성(코드/데이터/README): 10

통과 기준: 75점 이상

---

## 6) 자주 나는 실수 & 교정법

1. **축을 임의로 잘라 과장**
   - 교정: 축 범위 선택 이유를 캡션에 명시
2. **차트는 있는데 메시지가 없음**
   - 교정: 모든 차트에 "한 줄 결론" 필수
3. **색을 장식으로만 사용**
   - 교정: 색은 범주 구분/강조 목적에만 사용
4. **한 화면에 너무 많은 정보**
   - 교정: 핵심 질문 1~2개 중심으로 압축
