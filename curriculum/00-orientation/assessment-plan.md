# Assessment Plan (평가 설계)

## A. 진단평가 (Orientation Day 1)
- Python 기초 문법 (20점)
- 통계 기초 개념 (20점)
- SQL 기초 질의 (20점)
- Git/GitHub 사용 (20점)
- 문제해결/문서화 습관 (20점)

총점 100점 / 기준:
- 0~39: 완전 입문
- 40~69: 입문~초중급
- 70~100: 초중급 이상

## B. 형성평가 (챕터별)
각 챕터 종료 시 4개 항목을 5점 척도로 평가 (총 20점):
1. 개념 이해
2. 실습 완성도
3. 설명 가능성(타인에게 설명)
4. 재현 가능성(다시 실행 가능)

통과 기준: 14/20 이상

## C. 총괄평가 (월말)
- 미니 프로젝트 완성 (40점)
- 코드 품질/구조 (20점)
- 문서/회고 품질 (20점)
- 재현성/배포 가능성 (20점)

통과 기준: 75/100 이상

## D. 재학습 규칙
- 형성평가 14점 미만: 해당 챕터 1회 재학습
- 총괄평가 75점 미만: 부족 항목 보완 후 재평가
